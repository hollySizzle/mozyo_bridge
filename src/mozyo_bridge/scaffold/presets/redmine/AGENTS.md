# AGENTS

この project は mozyo-bridge scaffold rules を使用する。

## セッション開始

1. 現在の working directory がこの project root またはその配下であることを確認する。
2. mozyo-bridge の central preset rules を読む:
   - `${rule_path}`
3. 以下の project-local context file が存在する場合は読む。
4. 非自明な作業を始める前に active な Redmine issue と journal gate を確認する。

## Preset

- ticket system: `${preset}`
- preset version: `${preset_version}`
- generated by: `mozyo-bridge ${mozyo_bridge_version}`

## Project-Local Context

- `README.md`
- `vibes/docs/specs/project-map.md`
- `vibes/docs/rules/agent-workflow.md`

## Guardrails

- `${rule_path}` を取得できない場合は、読んだふりをせず停止する。
- Redmine issue と journal state を durable な作業状態として扱う。pane message と chat message は通知にすぎない。
- Redmine gate lifecycle (start / progress / design consultation / implementation done / review request / review / close) は `${rule_path}` を正本とする。
- 標準の agent 間通知は `mozyo-bridge notify-* --issue <issue_id> --journal <journal_id>` を使い、journal を作ってから pane へ通知する。
- credential、token、個人情報を repository file や Redmine note に記録しない。
