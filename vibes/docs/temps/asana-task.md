# Asana Task Template

```markdown
## 目的

## 作業対象パス

## 成果物パス

## 参照規約

## 完了条件

## 禁止事項
```

## 記入方針

- 目的は、なぜこの task が必要かを 1 から 3 文で書く。
- 作業対象パスには、変更・確認する local path または repository path を書く。
- 成果物パスには、作成・更新される file、workflow、release artifact、または external URL を書く。
- 参照規約には、Notion 規約または `vibes/docs/` 配下の文書を明記する。
- 完了条件は、検証 command または確認可能な状態で書く。
- 禁止事項には、secret handling や scope 外作業を明記する。
